# Pre-Interview Checklist

## 1 Day Before

### Technical Review
- [ ] Complete one full mock interview (SQL + Python + behavioral)
- [ ] Review weak areas identified by progress tracker
- [ ] Re-read SQL cheatsheet -- focus on window functions, CTEs, JOINs
- [ ] Re-read Python cheatsheet -- focus on pandas groupby, merge, apply
- [ ] Walk through 2-3 system design scenarios out loud
- [ ] Review flashcards flagged as low-confidence

### Company Research
- [ ] Re-read tasq.ai research notes
- [ ] Review the job description one more time
- [ ] Prepare 3-5 thoughtful questions to ask the interviewer
- [ ] Review LinkedIn profiles of interviewers (if known)

### Logistics
- [ ] Confirm interview time and timezone
- [ ] Save the meeting link / dial-in details somewhere accessible
- [ ] Test video conferencing tool (Zoom / Teams / Google Meet)
- [ ] Charge laptop fully
- [ ] Set two alarms for 90 minutes before the interview

---

## Morning Of (2 Hours Before)

### Equipment Check
| Item              | Status | Notes                                      |
|-------------------|--------|--------------------------------------------|
| Camera            | [ ]    | Clean lens, check framing and lighting     |
| Microphone        | [ ]    | Test in a recording app, reduce background |
| Internet          | [ ]    | Run a speed test, have mobile hotspot ready|
| Headphones        | [ ]    | Fully charged if wireless                  |
| Second monitor    | [ ]    | Optional but helpful for coding            |

### Environment Setup
- [ ] Clean and tidy the visible background
- [ ] Good lighting on your face (window or desk lamp in front)
- [ ] Close unnecessary browser tabs and applications
- [ ] Silence phone and desktop notifications
- [ ] Place a glass of water within reach
- [ ] Put a "Do Not Disturb" sign on the door if needed

### IDE / Coding Setup
- [ ] Open your preferred code editor or SQL client
- [ ] Verify Python environment works: `python --version`
- [ ] Verify you can run a quick SQL query in SQLite
- [ ] Have a blank scratch file ready for live coding
- [ ] Increase terminal/editor font size for screen sharing

### Documents to Have Ready
- [ ] Resume (PDF open on screen or printed)
- [ ] This project's GitHub page (ready to share link)
- [ ] `interview-day/quick_review.md` open in a tab
- [ ] Notepad or paper for jotting notes during the interview
- [ ] List of prepared questions for the interviewer

---

## 30 Minutes Before

### Mental Preparation
- [ ] Do the 15-minute quick review (`interview-day/quick_review.md`)
- [ ] Review your STAR stories (2-3 key ones)
- [ ] Take 5 slow deep breaths
- [ ] Remind yourself: you prepared well, you belong here
- [ ] Stand up, stretch, move around for a couple of minutes
- [ ] Smile -- it changes your vocal tone even on a call

### Final Technical Warm-Up
- [ ] Write one quick SQL query from memory (e.g., ROW_NUMBER with PARTITION BY)
- [ ] Write one quick Python snippet from memory (e.g., groupby + agg)
- [ ] Confirm your screen sharing works

---

## During the Interview

### General
- [ ] Greet the interviewer warmly, use their name
- [ ] Listen carefully to the full question before answering
- [ ] Ask clarifying questions -- it shows thoroughness
- [ ] Think out loud -- narrate your approach before coding
- [ ] Manage your time -- check how long you have for each section

### For Coding Questions
- [ ] Restate the problem in your own words
- [ ] Discuss your approach before writing code
- [ ] Start with a brute-force solution, then optimize
- [ ] Talk through edge cases
- [ ] Test your solution with the sample data
- [ ] Mention time/space complexity if relevant

### For Behavioral Questions
- [ ] Use the STAR framework (Situation, Task, Action, Result)
- [ ] Keep answers to 2-3 minutes
- [ ] Be specific -- use numbers, names, concrete outcomes
- [ ] Show self-awareness when discussing failures

### For System Design Questions
- [ ] Clarify requirements and constraints first
- [ ] Start with a high-level architecture diagram
- [ ] Discuss trade-offs for each technology choice
- [ ] Address scalability, monitoring, and failure modes
- [ ] Mention relevant technologies from the job description

---

## After the Interview

### Within 1 Hour
- [ ] Write down every question you were asked (memory fades fast)
- [ ] Note which answers felt strong and which felt weak
- [ ] Record any names, projects, or details the interviewer mentioned

### Within 24 Hours
- [ ] Send a thank-you email to each interviewer
- [ ] Reference something specific from the conversation
- [ ] Reaffirm your interest in the role
- [ ] Keep it concise (3-5 sentences)

### Reflection
- [ ] Rate your performance honestly (1-10 for each section)
- [ ] Identify topics to study if there is a next round
- [ ] Update your prep system with any new question types encountered
